# 全民参保手机看板 — 指标汇总

> 数据范围：当前所选年度 × 当前登录用户权限范围内的组织单元（区/镇街/村居/网格）  
> 数据来源：`residents` 居民表 + `enterprises` 企业表

---

## 一、居民参保核心指标（`/api/metrics/core`）

| 指标名称 | 字段Key | 口径含义 | 计算方法 / 规则 | 单位 | 统计粒度 |
|---|---|---|---|---|---|
| 参保总目标 | `target_total` | 当年应参保人数总目标 | `max(人口总数 × 92%, 已参保总数)` | 人 | 当年度 × 权限范围 |
| 职工保目标 | `target_staff` | 职工参保目标 | `max(参保总目标 × 42%, 已参职工数)` | 人 | 同上 |
| 居民保目标 | `target_resident` | 居民参保目标 | `参保总目标 − 职工保目标`（不低于已参居民数）| 人 | 同上 |
| 已参保总数 | `done_total` | 当年实际已缴费（参保）人数 | `residents.this_year_paid = 1` 的行数 | 人 | 同上 |
| 已参保职工数 | `done_staff` | 已参保且类型为职工保 | `this_year_paid=1 AND this_year_type='职工保'` | 人 | 同上 |
| 已参保居民数 | `done_resident` | 已参保且类型为居民保 | `this_year_paid=1 AND this_year_type='居民保'` | 人 | 同上 |
| 参保完成率（总） | `done_rate` | 已参保 / 目标 | `done_total / target_total × 100%`，保留1位小数 | % | 同上 |
| 参保完成率（职工） | `done_staff_rate` | 职工参保进度 | `done_staff / target_staff × 100%` | % | 同上 |
| 参保完成率（居民） | `done_resident_rate` | 居民参保进度 | `done_resident / target_resident × 100%` | % | 同上 |
| 待参保缺口（总） | `gap_total` | 距目标还差多少人 | `max(target_total − done_total, 0)` | 人 | 同上 |
| 待参保缺口（职工） | `gap_staff` | 职工保缺口 | `max(target_staff − done_staff, 0)` | 人 | 同上 |
| 待参保缺口（居民） | `gap_resident` | 居民保缺口 | `max(target_resident − done_resident, 0)` | 人 | 同上 |
| 可动员总数 | `mobilizable_total` | 当前可触达并动员参保的人数 | `存量可动员 + 增量可动员`（见下两行） | 人 | 同上 |
| 存量可动员 | `mobilizable_stock` | 去年已参、今年未参且不属于确认流失的人 | `stock_change_type='可动员' AND this_year_paid=0 AND 非确认流失` | 人 | 同上 |
| 增量可动员 | `mobilizable_increment` | 去年未参、今年也未参但仍在辖区内的新增潜在对象 | `this_year_paid=0 AND last_year_local_paid=0 AND 非确认流失 AND (本区户籍 OR 本区居住)` | 人 | 同上 |
| 风险预警标志 | `risk_flag` | 可动员数不够覆盖居民保缺口时触发 | `mobilizable_total < gap_resident` | 布尔 | 同上 |

> **确认流失类型**（以下情况视为无法动员，从可动员中排除）：死亡 / 辖区外参保 / 停保 / 转职工保（含灵活就业参保）

## 二、年龄分层指标（`/api/metrics/age`）

按5个年龄段分别输出：**16岁及以下 / 16-30岁 / 31-45岁 / 46-60岁 / 60岁以上**

| 指标名称 | 字段Key | 口径含义 | 计算方法 | 单位 |
|---|---|---|---|---|
| 年龄段人数 | `count` | 该年龄段总人口 | 按 `residents.age` 分组计数 | 人 |
| 年龄段占比 | `share` | 该年龄段人口占总人口比例 | `count / 总人口 × 100%` | % |
| 参保率 | `insured_rate` | 该年龄段已参保比例 | `this_year_paid=1的人数 / count × 100%` | % |
| 男性人数 | `male` | 该年龄段男性人口 | `gender='男'` 的行数 | 人 |
| 女性人数 | `female` | 该年龄段女性人口 | `count − male` | 人 |

## 三、职工参保指标（`/api/metrics/staff`）

### 3.1 参保人员维度

| 指标名称 | 字段Key | 口径含义 | 计算方法 | 单位 |
|---|---|---|---|---|
| 职工参保总人数 | `staff_people_total` | 当年参保类型为职工保的人数 | `this_year_type='职工保'` 的行数 | 人 |
| 职工参保大类分布 | `staff_big` | 按职工参保大类统计人数 | 按 `staff_big_type` 分组计数（字典值：在职职工 / 退休人员 / 灵活就业等）| 人/类 |
| 职工参保细分分布 | `staff_detail` | 按职工参保细分类型统计人数 | 按 `staff_detail_type` 分组计数 | 人/类 |

### 3.2 参保单位维度

| 指标名称 | 字段Key | 口径含义 | 计算方法 | 单位 |
|---|---|---|---|---|
| 单位总数 | `units_total` | 辖区内企业/单位总数 | `enterprises` 表行数 | 家 |
| 已参保单位数 | `units_insured` | 当前参保的单位 | `enterprises.staff_insured=1` 的行数 | 家 |
| 未参保单位数 | `units_uninsured` | 当前未参保的单位 | `units_total − units_insured`（≥0）| 家 |
| 单位参保率 | `units_insured_rate` | 单位参保覆盖比例 | `units_insured / units_total × 100%` | % |
| 单位未参保率 | `units_uninsured_rate` | 单位未参保比例 | `units_uninsured / units_total × 100%` | % |
| 单位参保数月环比 | `units_insured_mom` | 本月 vs 上月参保单位变化幅度 | `(本月参保单位 − 上月参保单位) / 上月参保单位 × 100%` | % |
| 单位未参保数月环比 | `units_uninsured_mom` | 本月 vs 上月未参保单位变化幅度 | `(本月未参保 − 上月未参保) / 上月未参保 × 100%` | % |

> 上月数据来源于 `enterprises.last_month_staff_insured` 字段

## 四、风险预警指标（`/api/metrics/risk`）

| 指标名称 | 字段Key | 口径含义 | 计算方法 | 单位 |
|---|---|---|---|---|
| 高风险企业数 | `high_risk_enterprises` | 当前评级为"高"风险的企业 | `enterprises.risk='高'` 的行数 | 家 |
| 中风险企业数 | `mid_risk_enterprises` | 当前评级为"中"风险的企业 | `enterprises.risk='中'` 的行数 | 家 |
| 转居民保人数 | `pause_flows.转居民保` | 停保流向为转居民保的居民 | `residents.pause_flow='转居民保'` 的行数 | 人 |
| 申请停保人数 | `pause_flows.申请停保` | 申请停保的居民 | `residents.pause_flow='申请停保'` 的行数 | 人 |
| 跨区转出人数 | `pause_flows.跨区转出` | 迁移到其他区参保的居民 | `residents.pause_flow='跨区转出'` 的行数 | 人 |

## 五、居民原始数据字段（`residents` 表）

> 对应前端「居民档案」列表，每一条记录为一名居民某年度的参保情况

| 字段名 | 中文含义 | 类型 | 枚举值 / 说明 |
|---|---|---|---|
| `id` | 居民唯一标识 | 文本 | 系统生成 UUID |
| `name` | 姓名 | 文本 | — |
| `phone` | 联系电话 | 文本 | 前端默认隐藏中间4位 |
| `gender` | 性别 | 文本 | 男 / 女 |
| `age` | 年龄 | 整数 | — |
| `unit_id` | 所属组织单元 | 文本 | 关联 org_units.id |
| `household` | 户籍状态 | 文本 | 本区户籍 / 外区户籍 / 港澳台 / 外籍 |
| `residence` | 居住状态 | 文本 | 本区居住 / 市内其他区 / 市外 |
| `residence_detail` | 居住细分 | 文本 | 固定居住 / 流动居住 / 其他 |
| `insured_place` | 参保地 | 文本 | 本区 / 外区 / 未参保 |
| `this_year_type` | 今年参保类型 | 文本 | 职工保 / 居民保 / 未参保 |
| `this_year_paid` | 今年是否已缴费 | 整数 | 1=已参保，0=未参保 |
| `last_year_paid` | 去年是否参保 | 整数 | 1=是，0=否 |
| `last_year_local_paid` | 去年是否在本区参保 | 整数 | 1=是，0=否 |
| `stock_change_type` | 存量变动类型 | 文本 | 可动员 / 死亡 / 辖区外参保 / 停保 / 转职工保（含灵活就业）|
| `loss_reason` | 流失原因 | 文本 | 死亡 / 辖区外参保 / 停保 / 转职工保（含灵活就业）/ 空 |
| `pause_flow` | 停保流向 | 文本 | 转居民保 / 申请停保 / 跨区转出 / 空 |
| `key_group` | 重点人群类型 | 文本 | 低保对象 / 残疾人 / 新生儿 / 老年人 / 其他 / 空 |
| `is_hardship` | 是否困难群体 | 整数 | 1=是，0=否 |
| `hardship_type` | 困难类型 | 文本 | 低保 / 低收入 / 特困 / 空 |
| `staff_big_type` | 职工参保大类 | 文本 | 在职职工 / 退休人员 / 灵活就业 / 空 |
| `staff_detail_type` | 职工参保细分 | 文本 | 各细分类别 / 空（参考字典表）|
| `household_addr` | 户籍地址 | 文本 | — |
| `residence_addr` | 居住地址 | 文本 | — |
| `year` | 数据年度 | 整数 | 如：2025 |

## 六、企业原始数据字段（`enterprises` 表）

> 对应前端「企业档案」列表，每一条记录为一家企业某年度的参保情况

| 字段名 | 中文含义 | 类型 | 说明 |
|---|---|---|---|
| `id` | 企业唯一标识 | 文本 | 系统生成 UUID |
| `name` | 企业名称 | 文本 | — |
| `legal_person` | 法定代表人 | 文本 | — |
| `contact_person` | 联系人 | 文本 | — |
| `phone` | 联系电话 | 文本 | 前端默认隐藏中间4位 |
| `address` | 企业地址 | 文本 | — |
| `unit_id` | 所属组织单元 | 文本 | 关联 org_units.id |
| `risk` | 风险等级 | 文本 | 高 / 中 / 低 |
| `staff_insured` | 当前是否参保 | 整数 | 1=已参保，0=未参保 |
| `last_month_staff_insured` | 上月是否参保 | 整数 | 1=是，0=否（用于月环比计算）|
| `gap_rate` | 参保缺口率 | 浮点 | 实际参保人数 vs 应参保人数的差距比例 |
| `duration` | 连续参保时长 | 整数 | 月数 |
| `year` | 数据年度 | 整数 | 如：2025 |
