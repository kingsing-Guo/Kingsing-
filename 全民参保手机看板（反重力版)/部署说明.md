# 全民参保手机看板 —— 部署说明

> 适用版本：反重力版 | 更新日期：2026-03-03  
> 支持数据库：**SQLite**（默认，本地验证）/ **MySQL 8.0+**（正式部署）

---

## 一、环境要求

| 依赖项 | 要求 | 说明 |
|---|---|---|
| Python | 3.7 + | 后端运行环境，macOS/Linux 已内置 |
| SQLite | 内置 | Python 自带，**本地验证无需额外安装** |
| mysql-connector-python | 8.0 + | 仅 MySQL 模式需要：`pip3 install mysql-connector-python` |
| MySQL | 8.0 + | 仅正式部署需要 |
| Nginx | 可选 | 生产环境推荐，统一入口 + 反向代理 |

## 二、数据库模式切换（核心）

所有切换通过 `backend/.env` 中的 **`DB_ENGINE`** 控制，其余代码文件**无需任何改动**。

```bash
# 首次配置
cp backend/.env.example backend/.env
# 然后编辑 backend/.env
```

### ▶ SQLite 模式（本地验证 / 演示，默认）

```ini
# backend/.env
DB_ENGINE=sqlite
DASHBOARD_SECRET=local-dev-secret-change-in-production
```

```bash
cd backend
python3 seed_db.py          # 生成 backend/data/dashboard.db（约 1.2 MB）
# 输出 "seed ok: .../dashboard.db" 表示成功
```

### ▶ MySQL 模式（服务器正式部署）

**Step 1：在 MySQL 中建库建用户（一次性）**

```sql
CREATE DATABASE dashboard
    CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'dashboard_user'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON dashboard.* TO 'dashboard_user'@'%';
FLUSH PRIVILEGES;
```

**Step 2：配置 `.env`**

```ini
# backend/.env
DB_ENGINE=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=dashboard_user
DB_PASSWORD=your_password
DB_NAME=dashboard
DASHBOARD_SECRET=your-strong-production-secret
```

**Step 3：安装驱动并初始化**

```bash
pip3 install mysql-connector-python
cd backend
python3 seed_db.py          # 在 MySQL 中建表并写入初始数据
```

> **两种模式切换步骤：** 修改 `.env` 中的 `DB_ENGINE` → 执行 `python3 seed_db.py` → 重启后端

## 三、本地快速运行

```bash
# 项目根目录
python3 start.py
```

启动后自动打开浏览器，并显示手机访问地址：

```
💻 本机访问：   http://127.0.0.1:8788/login.html?...

cd /opt/dashboard/backend

```

## 五、启动后端服务

### 方式 A — nohup（快速验证）

```bash
cd /opt/dashboard/backend
nohup python3 server.py > ../backend.log 2>&1 &
tail -f /opt/dashboard/backend.log    # 查看日志
```

### 方式 B — systemd（推荐生产，自动重启）

```bash
sudo nano /etc/systemd/system/dashboard-api.service
```

```ini
[Unit]
Description=全民参保看板后端 API
After=network.target

[Service]
WorkingDirectory=/opt/dashboard/backend
ExecStart=/usr/bin/python3 server.py
Restart=always
RestartSec=5
EnvironmentFile=/opt/dashboard/backend/.env

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable dashboard-api
sudo systemctl start dashboard-api
sudo systemctl status dashboard-api     # 查看状态
sudo journalctl -u dashboard-api -f     # 实时日志
```

## 六、配置 Nginx（推荐）

前后端同域访问，无跨域问题，无需在 URL 中带端口号。

```nginx
# /etc/nginx/conf.d/dashboard.conf
server {
    listen 80;
    server_name 服务器IP或域名;

    root /opt/dashboard;
    index login.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
```

访问地址：`http://服务器IP/login.html`（无需 `?apiBase=...`）

## 七、开放防火墙

```bash
# 使用 Nginx（只需开 80 端口）
sudo firewall-cmd --permanent --add-service=http && sudo firewall-cmd --reload

# 不用 Nginx 直接访问（开 8787 端口）
sudo firewall-cmd --permanent --add-port=8787/tcp && sudo firewall-cmd --reload
```

## 八、字典数据维护

字典枚举值（参保类型、重点人群等）存储在 `dictionaries` 表，直接改数据库即可，无需修改代码：

```sql
-- 新增字典值
INSERT INTO dictionaries(category, value, sort_order)
VALUES('重点对象', '老年人', 5);

-- 停用某个值（不删除，只是不展示）
UPDATE dictionaries SET enabled = 0
WHERE category = '居民居住细分' AND value = '市外';

-- 查看所有字典
SELECT * FROM dictionaries ORDER BY category, sort_order;
```

## 九、常见问题排查

| 问题 | 排查方向 |
|---|---|
| 后端报 `DASHBOARD_SECRET 未配置` | 检查 `backend/.env` 是否存在，`DASHBOARD_SECRET` 是否有值 |
| 后端报 `mysql-connector-python 未安装` | `pip3 install mysql-connector-python` |
| 后端报 `Can't connect to MySQL` | 检查 `DB_HOST/DB_PORT/DB_USER/DB_PASSWORD` 配置；确认 MySQL 服务已启动 |
| 登录失败 | 确认后端已启动；用 `curl http://IP:8787/api/health` 测试连通性 |
| 手机无法访问 | 确认与电脑/服务器在同一网络；路由器是否开启了 AP 隔离 |
| 切换 DB_ENGINE 后数据为空 | 切换后需重新执行 `python3 seed_db.py` 在新数据库中建表写入数据 |
| 切换年度后数据为空 | 数据库中无该年度的居民数据，需导入对应年度数据 |
