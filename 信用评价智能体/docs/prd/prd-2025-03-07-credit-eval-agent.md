## 一、产品概述

### 1.1 产品名称

**公共信用评价模型构建智能体**（简称：信用评价智能体）

### 1.2 产品定位

面向政府行业主管部门的 **AI 辅助信用评价模型构建工具**，通过对话式交互 + 可视化展示的方式，帮助业务人员高效、科学地完成信用评价模型的构建与验证工作。

### 1.3 目标用户

政府各行业主管部门的业务人员，包括但不限于：
- 交通局、环保局、城管局、民政局、经信局、科技局等

### 1.4 评价对象

行业监管主体：
- **主要**：企业法人
- **次要**：社会组织

### 1.5 产品价值

| 维度 | 传统模式 | 智能体模式 |
|------|----------|------------|
| 构建效率 | 需要领域专家、反复讨论、查阅大量资料 | AI 辅助生成，大幅缩短构建周期 |
| 专业性 | 依赖专家经验 | AI 结合政策文件、行业案例智能推荐 |
| 科学性 | 主要靠人工判断 | 统计学方法验证 + AI 分析优化建议 |
| 操作门槛 | 需要专业知识和经验 | 自然语言交互，降低使用门槛 |

### 1.6 项目边界

| 范围 | 说明 |
|------|------|
| 包含 | 模型构建、模型验证（试算）、模型优化、版本管理、模型发布、**行业信用评价管理办法文档生成** |
| 不包含 | 周期性评价执行、评价结果管理（由现有 evaluate-mamager 系统承担） |

## 二、信用评价模型结构

### 2.1 三大核心要素

#### 2.1.1 指标体系

- 多级树形结构：一级指标 - 二级指标 - 三级指标
- 每个指标代表评价主体的一个维度或角度
- 通过完整的指标体系形成全面的评价维度覆盖

#### 2.1.2 权重体系

- 每个指标在评价体系中的占比（百分比 / 分值）
- 权重决定了每个指标对最终评价结果的影响程度
- 同级指标权重之和应为 100%

#### 2.1.3 计算规则

每个末级指标（叶子节点）需要定义：

| 属性 | 说明 |
|------|------|
| **数据来源** | 从信用目录/数据目录选取，或由部门上传专有数据 |
| **评分算法** | 定义如何将原始数据转换为指标得分 |

**常见评分算法**：

| 算法类型 | 说明 | 示例 |
|----------|------|------|
| 区间评分法 | 将数据划分为若干区间，对应不同分值 | 注册资本 0-100万60分，100-500万80分，500万以上100分 |
| 阈值评分法 | 设定阈值，达标得满分，不达标得0分或扣分 | 有行政处罚记录扣20分 |
| 线性评分法 | 数据值与得分呈线性关系 | 纳税额每增加10万得分增加5分 |
| 比例评分法 | 按比例折算 | 实际值/目标值 x 满分 |
| 累计计分法 | 按事件数量累计计分 | 每获得一项荣誉+5分，上限20分 |

### 2.2 评价结果配置

评价结果采用 **分数制 + 等级制** 两者结合的方式：

#### 2.2.1 分数制

| 选项 | 说明 |
|------|------|
| **百分制** | 满分 100 分 |
| **千分制** | 满分 1000 分 |

用户在创建模型时可选择计分制度。

#### 2.2.2 等级制

用户可选择等级数量和等级名称：

| 等级模式 | 示例 |
|----------|------|
| **四等级制** | A（优秀）、B（良好）、C（一般）、D（较差） |
| **五等级制** | A（优秀）、B（良好）、C（中等）、D（较差）、E（差） |

#### 2.2.3 分数-等级映射

用户可自定义分数与等级的对应关系，例如（百分制四等级）：

| 等级 | 分数区间 |
|------|----------|
| A（优秀） | 90-100 分 |
| B（良好） | 75-89 分 |
| C（一般） | 60-74 分 |
| D（较差） | 0-59 分 |

### 2.3 公共信用综合评价约束（国标要求）

依据国家标准 **《公共信用综合评价规范》（GB/T 45255-2025）**，所有行业信用评价模型必须纳入公共信用综合评价结果作为评价因子之一。

| 约束规则 | 说明 |
|----------|------|
| **必须纳入** | 行业信用评价模型必须包含"公共信用综合评价结果"作为一个评价维度 |
| **最低权重** | 公共信用综合评价结果在行业评价模型中的权重**原则上不低于 10%** |
| **权重可调** | 用户可以设置大于等于 10% 的具体权重值 |
| **模型校验** | 模型构建时系统需校验该权重是否满足大于等于10%的要求 |

模型结构示意：

```
行业信用评价模型
  |-- 公共信用综合评价结果（权重 >= 10%，国标 GB/T 45255-2025 强制要求）
  |-- 行业专项指标（权重 <= 90%）
       |-- 一级指标 A
       |    |-- 二级指标 A1
       |    |-- 二级指标 A2
       |-- 一级指标 B
       |    |-- 二级指标 B1
       |    |-- ...
       |-- ...
```

### 2.4 数据来源

**主要来源（已有）**：
- 信用目录
- 数据目录
- 信用数据湖仓（主体数据库、专题数据库）

**补充来源（部门自有）**：
- 行业专有数据上传
- 未被采集治理的专业数据

## 三、智能体核心工作流程

### 3.1 整体流程（7个阶段）

```
Phase 1: 需求收集
    |-- 收集政策文件（用户上传，支持查看原文+AI解析摘要）
    |-- 搜索参考案例（其他地区类似行业的评价办法/指标）
    |-- 收集用户偏好与要求（关注重点、特殊需求）
         |
         v
Phase 2: 智能构建
    |-- AI 基于材料智能生成初始模型
    |    |-- 生成指标体系（含综合评价权重）
    |    |-- 推荐权重分配
    |    |-- 建议计算规则
    |-- 展示完整模型供用户查看
         |
         v
Phase 3: 人工调整
    |-- 用户查看模型细节
    |-- 可视化编辑指标/权重/计算规则
    |-- 配置分数制、等级制、分数-等级映射
    |-- 用户确认或提出修改意见
         |
         v
Phase 4: 模型验算
    |-- 选择样本企业（用户提供或平台随机选取）
    |-- 运行模型试算
    |-- 输出试算结果
         |
         v
Phase 5: 结果评估与优化（迭代循环）
    |-- 统计学分析（K-S 区分度、分布度等）
    |-- AI 给出分析报告和优化建议
    |-- 用户确认优化方向
    |-- 智能体执行优化
    |-- 重新验算，循环直到满意
         |
         v
Phase 6: 版本保存与发布
    |-- 保存为模型版本（含版本号、描述、快照）
    |-- 通过 API 发布模型到评价执行系统
         |
         v
Phase 7: 管理办法生成
    |-- 基于已发布的模型 + 参考模板
    |-- AI 生成标准公文格式的行业信用评价管理办法
    |-- 用户查看、编辑、下载 Word 文档
         |
         v
Phase 8: 知识沉淀与入库 (数据飞轮)
    |-- 系统对已发布的优秀模型进行脱敏拆解
    |-- 将优质的“评价指标”、“评分规则”及“管理办法”补充至平台公共字典与向量知识库
    |-- 反哺未来的智能构建与案例检索 (RAG 增强)
```

### 3.2 各阶段详细说明

#### Phase 1：需求收集

| 功能点 | 说明 |
|--------|------|
| **政策文件上传** | 用户上传上级指导文件、管理办法等 PDF/Word 文档；支持查看原文、AI 全文解析和内容摘要 |
| **参考案例搜索** | AI 搜索其他地区已发布的类似行业信用评价管理办法和评价指标 |
| **需求偏好收集** | 通过对话方式引导用户说明模型关注重点（如科创能力、经营风险、合规表现等） |
| **需求汇总确认** | 在展示区汇总用户提供的所有材料和需求，供用户确认和补充 |

#### Phase 2：智能构建

| 功能点 | 说明 |
|--------|------|
| **指标体系生成** | AI 基于收集的材料，智能生成多级指标体系（自动纳入综合评价权重） |
| **权重推荐** | AI 根据政策文件要求和行业特点推荐各指标权重 |
| **计算规则建议** | AI 为每个末级指标推荐合适的数据来源和评分算法 |
| **模型完整展示** | 在展示区以结构化方式呈现完整模型 |

#### Phase 3：人工调整

| 功能点 | 说明 |
|--------|------|
| **指标编辑** | 支持增删改指标，调整层级关系 |
| **权重调整** | 可视化调整各指标权重，实时校验合计值 |
| **综合评价权重** | 设置公共信用综合评价结果的权重（系统校验不低于10%） |
| **评价结果配置** | 选择分数制（百分制/千分制）、等级制（四级/五级）、配置分数-等级映射 |
| **计算规则修改** | 修改数据来源选择、评分算法配置 |
| **对话式修改** | 用户也可以通过自然语言描述修改需求，AI 执行修改 |

#### Phase 4：模型验算

| 功能点 | 说明 |
|--------|------|
| **样本选择** | 用户手动指定样本企业，或从主体库随机抽取 |
| **试算执行** | 基于当前模型配置，对样本企业运行评价计算 |
| **结果展示** | 展示每个样本企业的评价得分、等级及各指标明细 |

#### Phase 5：结果评估与优化

| 功能点 | 说明 |
|--------|------|
| **统计学分析** | K-S 区分度检验、分数分布直方图、等级分布比例等 |
| **AI 分析报告** | 智能体对试算结果进行综合分析，判断模型科学性和合理性 |
| **优化建议** | 具体指出应优化指标/权重/计算规则中的哪些部分，并给出建议值 |
| **交互式优化** | 用户在对话中确认优化方向，智能体自动执行优化 |
| **迭代循环** | 优化后重新验算，直到用户满意 |

| 功能点 | 说明 |
|--------|------|
| **版本管理** | 多版本随时对比、恢复，记录版本修改日志 |
| **执行系统发布** | 一键将经过验证的评价模型发布对接至 `evaluate-manager` 执行系统使用 |

#### Phase 7：管理办法生成及 Phase 8：知识沉淀

| 功能点 | 说明 |
|--------|------|
| **公文生成** | AI 基于确定后的指标体系和计算权重规则，渲染生成符合政府标准的红头《管理办法》说明文档 |
| **知识反哺入库** | 抽取脱敏发布后的优质业务评价指标、权重经验及公文模板，追加挂载至通用大模型 Vector(RAG) 数据库中 |

### 3.3 信用评价专业知识库 (RAG) 的建设与演进机制

不仅作为**效率工具**，本智能体平台的定位同时是**信用评价核心智库**。为了赋能从“人工搭建”走向“数据飞轮体系”，我们将构建一套基于 RAG（检索增强生成）和向量化搜索体系的公共信用专业知识库：

1. **知识库初始构成（冷启动库）**：
   - **国家及地方政策库**：如《公共信用综合评价规范》(GB/T 45255-2025)、各省市优良的示范性管理办法。
   - **基础指标字典库**：发改委或平台预置的跨行业通用考核维度（环保、社保、税务、安全生产等）结构与计算标准。
   - **算法算子库**：区间计分、阈值计分、非线性折减等标准化评分算子。

2. **知识积累与沉淀闭环（数据飞轮）**：
   - 各委办局在智能体平台上通过大量模拟、推演和最终发布实际业务模型后，系统将自动识别高频共性指标（例如多部门均抓取了“行政处罚次数”并赋予一定权重）。
   - 用户可主动将自己打磨好的行业模型，一键“提交入库”至平台级经验库中。
   - 智能系统对入库模型进行脱敏、标签化和 Embedding 抽取操作，转变为日后其他用户输入提示词时可被大模型“精准召回 (Retrieval)”的养料。

3. **智能应用表现**：
   - 当新用户提出“要求建一个针对网约车司机的评级模型”时，AI 大模型将在自己的预训练基础智力外，通过 RAG 挂载**本评级平台的知识库**。优先推荐该知识库中同行验证过的顶级指标字典，确保专业度和合规度。
|--------|------|
| **版本保存** | 保存当前模型为一个版本（版本号、描述、完整模型快照） |
| **版本历史** | 查看模型的所有历史版本 |
| **版本对比** | 对比不同版本的差异 |
| **模型发布** | 通过 API 将确认的模型版本推送到评价执行系统 |

#### Phase 7：管理办法生成（模型发布后）

| 功能点 | 说明 |
|--------|------|
| **模板选择** | 提供标准公文格式的管理办法参考模板 |
| **AI 智能生成** | 基于已发布的评价模型（指标体系、权重、计算规则）+ 参考模板，AI 自动生成行业信用评价管理办法文档 |
| **文档内容** | 包含总则、评价对象与范围、评价指标与权重、评分方法、等级划分、结果应用、附则等标准章节 |
| **在线编辑** | 用户可在展示区查看和编辑生成的管理办法内容 |
| **文档下载** | 支持导出为 Word（.docx）格式的标准公文文档 |

## 四、界面设计

### 4.1 整体布局

双栏布局：
- **左侧 - 对话交互区**（约 40% 宽度）：用户与智能体的对话窗口
- **右侧 - 过程和结果展示区**（约 60% 宽度）：根据当前阶段动态展示内容
- **顶部导航栏**：Logo / 项目名称 / 当前部门 / 用户信息
- **底部状态栏**：当前阶段 / 模型状态 / 版本

### 4.2 对话交互区

| 功能 | 说明 |
|------|------|
| **消息列表** | 展示用户和智能体的对话历史，支持富文本、表格等格式 |
| **文件上传** | 支持上传政策文件（PDF、Word、Excel） |
| **文件预览** | 支持查看已上传文件原文 + AI 摘要 |
| **文本输入** | 自然语言输入框，支持多行输入 |
| **快捷操作** | 预设常用指令按钮（如"开始构建"、"运行验算"、"优化模型"等） |
| **对话持久化** | 每个模型的对话历史持久化保存，可随时查看 |

### 4.3 过程和结果展示区

根据当前构建阶段，动态渲染不同的展示面板：

| 阶段 | 展示内容 |
|------|----------|
| 需求收集阶段 | 需求汇总卡片：已上传的文件列表（支持点击查看原文与AI摘要）、收集到的参考案例、用户需求偏好摘要 |
| 模型构建阶段 | 模型结构展示：指标树形图（含综合评价权重标识）、权重饼图/柱状图、计算规则配置表、分数制/等级制配置 |
| 人工调整阶段 | 可编辑的模型面板：支持拖拽排序、权重滑块调整、规则编辑、分数-等级映射配置 |
| 模型验算阶段 | 验算结果：企业评分列表、得分分布图、各指标得分明细、等级分布 |
| 结果评估阶段 | 分析报告：K-S 检验图、分布直方图、AI 分析文字、优化建议列表 |
| 版本管理阶段 | 版本列表：版本号、创建时间、描述、操作（查看/发布/对比） |
| 管理办法生成阶段 | 管理办法文档：标准公文格式预览、在线编辑、Word 下载 |

## 八、已确认问题

| 编号 | 问题 | 确认结果 |
|------|------|----------|
| Q1 | 评价结果是分数制还是等级制？ | **两者结合**：分数制支持百分制和千分制；等级制支持四级和五级；用户可自定义分数与等级的映射关系 |
| Q2 | 模型发布后如何与现有系统对接？ | **通过 API 推送** |
| Q3 | 是否需要模型模板库？ | **需要**，同时新增国标约束：依据 GB/T 45255-2025，行业评价模型必须纳入公共信用综合评价结果（权重>=10%） |
| Q4 | 对话历史是否需要持久化？ | **每个模型的对话历史需要持久化保存** |
| Q5 | 是否需要多人协作？ | **不需要**，每个模型一个负责人 |
| Q6 | 上传文件是否需要解析？ | **需要**：支持查看原文 + AI 全文解析 + 内容摘要 |
| Q7 | 模型发布后是否有后续环节？ | **新增**：AI 生成标准公文格式的行业信用评价管理办法，支持编辑和 Word 导出 |

## 九、术语表

| 术语 | 解释 |
|------|------|
| 信用评价模型 | 由指标体系、权重体系、计算规则三个核心要素组成的评价体系 |
| 指标体系 | 多层级的评价维度集合（一级到二级到三级指标） |
| 权重 | 各指标在评价中的占比 |
| 计算规则 | 指标数据来源和评分算法的定义 |
| 信用目录 | 已有的公共信用数据分类目录系统 |
| 数据目录 | 已有的公共信用数据资源目录系统 |
| 信用数据湖仓 | 已采集治理的公共信用数据存储系统 |
| K-S 检验 | Kolmogorov-Smirnov 检验，用于评估模型区分度 |
| 模型验算 | 使用样本企业数据对模型进行试运算，验证模型的合理性 |
| GB/T 45255-2025 | 《公共信用综合评价规范》国家标准，要求行业信用评价纳入综合评价结果 |
| 公共信用综合评价 | 国家规定的基础信用评价体系，行业评价需以此为基础（权重>=10%） |
| 行业信用评价管理办法 | 各行业主管部门制定的信用评价管理规范性文件（标准公文格式） |
| 分数-等级映射 | 将评价分数映射为信用等级的对应关系规则 |
